| 구분 | 1차 페르소나 | 2차 페르소나 |

| 이름 | WhaleScope 운영자 (On-call) | 뤼튼 과제 심사위원 |

## 3. Emotional Tone

- **1차 감정**: Trust (신뢰) — 데이터·출처·업데이트 시간이 투명
- **2차 감정**: Calm (평온) — 위기 상황(시그널 BAD)이 있어도 UI가 흔들리지 않음
- **금지 감정**: 긴박(Urgency·Alarm), 과잉 딜라이트(Playful)

운영 대시보드는 긴박함이 무기가 아니라 **흔들리지 않는 차분함**이 무기. 시그널이 터졌을 때 대시보드가 평온해야 사용자가 판단을 할 수 있다.

## 4. Differentiator (단 하나 기억될 요소)

> **"매일 아침 읽는 한 장의 레터 — Daily Brief"**

- `brief-card`를 대시보드의 **히어로 포지션**으로 배치
- 에디토리얼 타이포그래피 (Manrope Display + Pretendard Body, 큰 제목, 여유로운 행간 1.6+)
- 본문 너비 **65~75자 / 줄**로 제약 — 뉴스레터 독서감
- 다른 데이터 위젯은 brief를 **보좌하는 사이드바**로 시각적 명도·무게를 의도적으로 낮춤
- 인용·강조·출처가 **한 단락 안에 자연스럽게 삽입** — 데이터 대시보드라기보다 읽을 것이 있는 뉴스 섹션

## 6. Tone 보드 (극단 선택 — "미지근함 금지")

- [x] **Editorial-calm**: 큰 여백 + 고요한 블루 + 절제된 강조
- [ ] Brutalist, Maximalist, Retro-futuristic, Playful, Organic, Luxury, Industrial

한 축만 선택하고 그 강도로 실행. "일반적으로 깔끔한" 중간 지점을 피함.

## 7. Guardrail — 절대 하지 않는 것

- 보라·핑크 그라디언트 배경 (AI slop 표식)
- 그라디언트 텍스트 (`background-clip: text` 장식)
- 이모지 UI 아이콘 → Material Symbols Outlined 또는 Lucide만 허용
- `transition: all` → 반드시 속성 지정
- `outline: none` without `focus-visible` 대체
- `<div onClick>` / `<span onClick>` → `<button>`·`<a>` 사용
- 5종 이상 폰트 사이즈를 한 화면에 혼재
- 8pt 그리드 이탈 값 (13px, 17px, 25px 등)

## 9. 유지보수 규칙

1. **Layer 1 결정을 바꿀 때만 이 파일을 수정**. 토큰 값 조정은 `DESIGN.md`.
2. 새 화면·컴포넌트 작업 시 Claude는 `.design-context.md` → `DESIGN.md` → `design-tokens.css` 순서로 읽는다.
3. 새 페르소나·톤·차별화를 추가할 때는 **기존 것을 폐기할지 병치할지** 명시.
4. Tone 변경은 곧 시각 전체의 재조정이므로 가볍게 하지 않는다.
