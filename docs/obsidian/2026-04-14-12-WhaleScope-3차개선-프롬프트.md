## 공통 규칙 (모든 Track 실행 시 적용)

```
- 작업 시작 전 `git status` 확인. 클린 상태가 아니면 먼저 보고하고 중단.
- 각 Track은 별도 브랜치에서 작업: feature/fix-p0-b1, feature/fix-p0-b2, ...
- 하나의 Track은 하나의 논리적 관심사만 다룬다. Track 간 파일이 겹치면 순차 실행.
- 커밋은 Track 내부에서 의미 단위로 분리. squash 금지.
- 각 Track 완료 시 `pytest -q --tb=line`으로 전체 테스트 통과 확인 후 종료.
- 테스트가 깨지면 해당 Track 내에서 수정. 다음 Track으로 넘어가지 않는다.
- README 또는 CHANGELOG에 변경 요약 추가(bullet 한두 줄).
- .env, credentials, API 키를 코드·커밋·로그에 절대 포함하지 않는다.
- 기존 Sheets 헤더 순서를 바꾸지 않는다. 추가만 가능.
```
