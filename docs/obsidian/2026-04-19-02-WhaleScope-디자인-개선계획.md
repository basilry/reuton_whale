| Layer | 점수 | 한 줄 평 |
|---|---|---|

### 핵심 발견 5가지

1. **시스템(Layer 2)은 거의 만점** — 새로 만들 필요 없다. 가이드를 더 잘 "쓰기"가 과제다.
2. **AI Slop 0건** — `transition: all`·gradient text·side-stripe(4px+)·purple-pink 그라디언트·UI ease-in 모두 발견되지 않음. 이미 design-engineering 가이드를 알고 작성된 코드.
3. **시각적 단조** — 메인 페이지의 모든 카드(brief, mood, signal, watchlist, story, news, ticker)가 동일한 `panel-glass + blur 20px + radius-lg + elev-2~3` 조합. **무엇이 hero인지 시각적으로 구분되지 않는다**. "모든 게 똑같이 중요해 보이면 아무것도 중요해 보이지 않는다"는 design-critique의 1번 죄.
4. **빈 상태가 곧 첫인상** — `EmptyBrief("구조화된 하이라이트 없음")`처럼 사용자에게 "이 서비스 비어 있음"이라는 메시지가 노출되는 슬롯이 다수. 첫 방문자 온보딩 임팩트 ↓.
5. **whale-story-panel.tsx가 자기 module.css 없이 `whale-story-detail-modal.module.css`를 import** — 컴포넌트–스타일 결합이 깨진 안티패턴. 리팩토링 타깃.

### 본 계획의 위치

본 계획은 [[2026-04-19-03-WhaleScope-장애대응-및-개선-이행계획]] §13 이행 순서의 **M2 트랙 D(Design Polish)** 로 편입을 권장한다. P1-H(공포탐욕지수)와 P1-G(SSE 실시간) 등 기능 트랙과 충돌 없이 병렬 진행 가능하다. M0–M1 운영 안정화 이후, M2 시작 시점에 본 §3·§4 우선 적용.

### 1.2 Layer 1 질문 3개 답안 (현 상태 vs 권장)

#### Q1. Who & Why — 누가 쓰고 어떤 마찰을 없애려는가?

| 항목 | 현 상태 | 권장 보강 |
|---|---|---|
| 대상 페르소나 | "user" — 추상적 | **"15분마다 X·CT를 새로 고치며 고래 움직임을 따라잡으려는 리테일 트레이더"** 로 구체화 |
| 1차 잡(Job-To-Be-Done) | "휴먼-리더블 브리프 제공" | **"오늘 아침/지금 이 순간, 시장 분위기와 상위 고래 1~3건의 의미를 30초 내 흡수"** |
| 2차 잡 | "큐레이션 지갑·라이브 마켓 컨텍스트" | "관찰 지갑 1~2개를 10초 내 추가 / 텔레그램으로 핸드오프" |
| 누락된 "Why now" | 없음 | **"왜 지금 이 화면을 봐야 하는가" — 마지막 갱신 시간·다음 갱신 시간·현재 시장 분위기를 화면 상단에서 즉시 인지** |

#### Q2. Emotional Tone — 어떤 감정을 심을 것인가?

현재 톤: **신뢰 + 평온 (Editorial-calm)**.
정합성: ✅ 토큰·타이포·여백·glass panel 모두 이 톤과 일치.
**그러나** — 임팩트가 부족한 영역에서는 톤이 *blandness*(무미건조)로 미끄러진다. 두 가지 보강 필요:

- **확신(Conviction)** 의 한 모금 — Hero brief의 첫 두 문장에 "지금 의미 있는 한 가지" 라는 강한 어조의 lead를 명시. 현재는 운영 메시지(`brief.summary`)가 그대로 노출되어 톤이 평준화돼 있음.
- **신호(Alertness)** 의 색채 — Critical 시그널이 발생했을 때, 카드 단독으로는 강조가 약함. Hero 영역에 **"⚠ 활성 critical 시그널 N건"** 의 sticky aura(테두리 1px → 2px + accent glow)를 부여.

#### Q3. Differentiator — 단 하나 기억될 요소

현재 시각적 차별점: **glassmorphism + 모든 카드의 `--panel-glass`** — 일관성은 있으나 *한 화면을 떠올렸을 때 기억에 남는 단 하나*가 없다.

권장: **"Hero brief의 시그니처 글로우 + 우측의 Fear&Greed 게이지"** 를 화면의 시그니처 페어로 만든다. P1-H로 추가되는 SVG semicircle 게이지가 유일한 *non-card visual element* 가 될 수 있다 — 이를 잘 활용하면 "WhaleScope는 게이지가 있는 그 대시보드" 라는 정체성을 만들 수 있다.

### 1.3 권장: `.design-context.md` 신규 생성

`apps/dashboard/.design-context.md` 파일을 추가해 위 결정을 영속화하고, 이후 Claude Code/Cowork 세션이 일관된 톤으로 작업하도록 한다.

```markdown
# WhaleScope · Design Context

## Audience
Retail crypto trader, 25–40, 하루 5~20회 모바일·데스크탑 양쪽으로 진입.
주된 마찰: "지금 시장이 어떤지, 내가 봐야 할 한 건이 무엇인지" 30초 내 파악.

## Tone
Editorial-calm · Light-blue · Glass — **신뢰 + 평온**을 기본으로,
critical 시그널·extreme F&G 구간에서만 **경계(alertness)** 를 짧게 발화한다.

## Differentiator
Hero brief 좌측 + Fear&Greed semicircle 우측의 시그니처 페어.
이 두 요소는 화면을 떠나도 기억되어야 한다.

## Non-goals
- 데이터 밀도 극대화(Bloomberg 류) → 추구하지 않음
- 다크모드를 디폴트로 → 추구하지 않음(브랜드는 light-first)
- 마이크로 모션 과다 → 운영 정보 화면이라 절제
```

## §2 Layer 2 — System & Tokens (Ground)

### 2.1 점수 9.5 / 10 — 모범 사례

`design-tokens.css`는 design-engineering 가이드의 권장 사항을 거의 100% 구현한다:

| 가이드 권장 | 구현 상태 |
|---|---|
| OKLCH (perceptually uniform) | ✅ 모든 색상 OKLCH, 60/30/10 비율 명시 |
| 8pt 스페이싱 ramp | ✅ `--space-3xs(2px) ~ --space-5xl(128px)` |
| 5단계 elevation | ✅ `--elev-0 ~ --elev-5` + blue-tinted shadow |
| 블루 틴트 shadow | ✅ `rgb(0 65 106 / 0.05)` 등 — pure black 회피 |
| ease-out / ease-drawer / ease-in-out | ✅ + `--ease-spring-soft`(rare delight 용) |
| no `ease-in` 정책 | ✅ 토큰 자체에 `Never: ease-in on UI` 주석 |
| dark theme + prefers-color-scheme + theme-boot | ✅ `data-theme="dark"` + `@media` 폴백 + JS pre-paint |
| prefers-reduced-motion | ✅ duration 토큰 0ms로 강제 + animation-duration 0.01ms |
| 폰트 페어링 (디스플레이 + 본문) | ✅ Manrope(display) + Pretendard(body) |
| 시멘틱 토큰 (--good/--warn/--bad) | ✅ + `-soft / -softer` 변형 + signal-* 파생 토큰 |

### 2.2 보강 4가지 (반쪽짜리 토큰화)

#### B-1. **타이포 스케일에서 누락된 슬롯**
`fear-greed-gauge.module.css:121` — `font-size: 2.5rem` 하드코딩.
현재 스케일에서 `--text-3xl(2.25rem)` 과 `--text-4xl(3rem)` 사이에 갭이 있어 디자이너가 임의로 `2.5rem`을 끼워 넣음.
**권장**: 토큰에 `--text-display-sm: 2.5rem; /* 40px — gauge value, KPI numerator */` 추가.

#### B-2. **Material Symbols vs Lucide React**
`globals.css:48` — `.material-symbols-outlined` 유틸리티 + `layout.tsx:54` 외부 폰트 CDN.
- 무거운 외부 폰트(Material Symbols Outlined 가변 폰트)는 첫 paint에 가산점이 없고, 사용자 시스템 환경에 의존.
- 사용자 환경에 `lucide-icons` 스킬이 있고, 본 가이드의 anti-pattern 11번이 **"이모지 대신 SVG 아이콘 - Lucide 권장"**.
**권장**: `lucide-react` 도입(번들 ~30KB, 트리쉐이킹) + `material-symbols-outlined` 점진적 대체. 새 컴포넌트(P1-H, P1-G)부터 Lucide 사용.

#### B-3. **`color-mix(in srgb, ...)` 의 유지보수성**
`fear-greed-gauge.module.css:36, 57, 142` 등에서 `color-mix(in srgb, var(--surface-container-high) 78%, transparent)` 사용 — 표현력은 좋지만, **반복 패턴**이 토큰화되지 않음.
**권장**: 자주 쓰는 색 mix 결과를 토큰으로 끌어올리기:
```css
--surface-container-glass: color-mix(in srgb, var(--surface-container-high) 78%, transparent);
--surface-container-glass-soft: color-mix(in srgb, var(--surface-container-high) 54%, transparent);
```

#### B-4. **radius 일관성 깨짐**
- `market-ticker-strip.module.css:13` → `--radius-xl (1.5rem / 24px)`
- `brief-panel`·`watchlistCard`·`heroCard` → `--radius-lg (1rem / 16px)`
- `signal-action-card` → `--radius-md (0.75rem / 12px)`
**문제**: 같은 위계의 카드(brief vs ticker)가 다른 둥글기. 시각적으로 ticker만 둥글둥글해 보임.
**권장**: 카드 위계 ↔ radius 매핑 룰을 토큰 주석에 명문화하고 ticker를 `--radius-lg` 로 통일.

### 2.3 토큰 거버넌스 룰 (신규)

`design-tokens.css` 상단 주석에 추가:

```css
/* GOVERNANCE
   1. 컴포넌트 CSS는 var(--token) 만 사용. hex/rgb/oklch 리터럴 금지.
   2. 부득이한 px 사용은 ≤4px (border, dot, line) 만 허용. 그 외는 spacing 토큰.
   3. radius 매핑:

        secondary card                     → --elev-2
        primary / hero card                → --elev-3
        popover / dropdown                 → --elev-4
        modal / sheet                      → --elev-5
*/
```

### 3.2 컴포넌트 횡단 패턴 결함

#### P-1. **모든 카드가 동일한 시각적 무게**

7개 주요 카드(`heroCard`, `moodCard`, `signalsPanel`, `watchlistCard`, `whale-story panel`, `news-widget`, `market-ticker-strip`)가 모두 다음 조합:

```css
background: var(--panel-glass);
backdrop-filter: blur(20px);
border: 1px solid var(--line);
border-radius: var(--radius-lg);
box-shadow: var(--elev-2~3);
```

**결과**: 한 화면에서 **시각적 위계가 사라지고**, 사용자가 "어디부터 봐야 할지" 모호해진다.

**개선 패턴 — 카드 위계 3-tier**:

| Tier | 용도 | 시각 처리 |
|---|---|---|
| **Hero** | 1개. brief + F&G 게이지 페어 | `--panel-strong (98% 불투명) + blur 24px + --elev-3 + --accent-soft 1px ring + radius-xl` + heroCardGlow 유지 |
| **Primary** | 2~3개. signal, watchlist, story | `--panel-glass + blur 20px + --elev-2 + --line 1px + radius-lg` (현재 디폴트) |
| **Secondary** | 다수. news, ticker, mini-cards | `--surface-container-low + blur 0 + --elev-1 + --line 1px + radius-md` (서브 정보임을 명시) |

#### P-2. **빈 상태 = 첫인상**

`brief-panel.tsx:120, 137, 159` 의 `EmptyBrief` 들:
> "구조화된 하이라이트 없음" / "테마 태그 없음" / "상위 거래 없음"

이 문구는 **운영자에게는 진단 정보**지만, **사용자에게는 "이 서비스는 비어 있다"는 시그널**.

**개선**: 빈 상태를 **"다음 갱신 예정 + 직전 데이터의 요약"** 구조로 재설계.
```tsx
// before
<EmptyBrief title="구조화된 하이라이트 없음" body="요약 본문은 생성됐지만…" />

// after — 사용자 페르소나 기준
<NextUpdateNotice
  title="다음 브리프 갱신 — 12분 후 (KST 14:00)"
  lastSummary="직전 브리프: BTC ETF 자금유입 둔화, ETH 빈자리 채움."
  cta={{ label: "이전 브리프 보기", onClick: ... }}
/>
```

#### P-3. **whale-story-panel.tsx의 스타일 차용 안티패턴**

```tsx
// whale-story-panel.tsx:8
import styles from "./whale-story-detail-modal.module.css";
```

자기 컴포넌트의 module.css가 없고, 모달의 CSS를 빌려 쓰고 있다. **규칙: 컴포넌트는 자기 스타일을 소유한다**.

**리팩토링 액션**:
1. `whale-story-panel.module.css` 신규 생성
2. `whale-story-detail-modal.module.css` 에서 `panel`, `storyButton`, `storyCard`, `storyDot`, `storyCopy`, `storyTitle`, `storyBody`, `storyMeta`, `storyAction`, `emptyState` 클래스 분리/이동
3. 모달은 자기 자신의 모달 전용 클래스만 보유

#### P-4. **숫자 컬럼의 `font-variant-numeric: tabular-nums` 누락 추정**

`market-ticker-strip`·`watchItem`·`signals-table`의 가격/수치 컬럼이 비례폭 숫자라면, 행간 정렬이 흔들린다.
**권장**: `--font-mono` 사용처 외에 **숫자 컬럼 전용 유틸리티 클래스** `tabularNums { font-variant-numeric: tabular-nums; }` 신설하고, 가격·시각·수치 셀에 일괄 적용.
