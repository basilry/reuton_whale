# WhaleScope 자체 수집 전환 계획 v2 — 하이브리드 소스 + 개인화 + 트렌드 코멘터리

v1 대비 변경점: Grok 제안(Whale Alert **공개 TG 채널** + Known Whale polling 하이브리드, 자연어 피드백 개인화, 주간 누적 트렌드 코멘터리)을 병합하고, 바뀌지 않은 블록(시그널 엔진, Sheets 스키마 일부)은 v1에서 재사용.

## 0. v1 대비 달라진 것

1. **Whale Alert 유료 API 완전 제거는 유지**. 다만 **공개 TG 채널 `@whale_alert_io`(318K subs, 공개)**을 "방송 수신기" 용도로 병용. 이건 API 라이선스·과금 없이 접근 가능한 공개 브로드캐스트라 "의존"이 아니라 "참조 신호".
2. 원본 데이터 두 축: **(A) Whale Alert TG 수신 + (B) Known Whale on-chain polling**. 교차검증으로 재현성↑, 커버리지↑.
3. Claude의 역할을 **이중 레이어**로 재정의: 규칙 기반 시그널 점수는 `SignalEngine`이, 시장 영향 해석·주간 트렌드 코멘터리는 Claude가. v1은 Claude를 해석자로만 제한했지만 v2는 "구조화된 내러티브 생성기"로 확장.
4. **자연어 개인화 루프**: `/watchlist BTC ETH` 같은 명령형이 아니라 "BTC 고래만 더 자주 보고 싶어요" 같은 자연어를 Claude가 해석해 `user_interests`에 가중치 저장.
5. **트렌드 코멘터리** 산출물 신설: 일간 브리프와 별도로 주간 "Net Accumulation" 요약.

## 1. 데이터 소스 아키텍처 (하이브리드)

```
          ┌──────────────────────────┐
Source A  │ Whale Alert public TG    │  (318K subscribers, 공개 broadcast)
          │  @whale_alert_io         │  → Telethon user-account listener
          └────────────┬─────────────┘
                       │
                       ▼
          ┌──────────────────────────┐     ┌──────────────────────────┐
          │  tg_whale_events (raw)   │     │  address_activity (raw)  │
          └────────────┬─────────────┘     └────────────┬─────────────┘
                       │                                ▲
                       │                                │
                       │         ┌──────────────────────┴────────┐
Source B               │         │ Etherscan / Solscan / BscScan │
                       │         │ polling (Known Whale list)    │
                       │         └──────────────────────┬────────┘
                       │                                │
                       └────────────┬───────────────────┘
                                    ▼
                          ┌──────────────────┐
                          │   Normalizer     │ (두 소스를 동일 스키마로)
                          └────────┬─────────┘
                                   ▼
                          ┌──────────────────┐
                          │  SignalEngine    │ (규칙 기반 스코어)
                          └────────┬─────────┘
                                   ▼
                          ┌──────────────────┐
                          │ Claude Analyzer  │ (해석 + 트렌드)
                          └────────┬─────────┘
                                   ▼
                         ┌────────────────────┐
                         │ Daily Brief / Week │
                         │ Trend / Telegram   │
                         └────────────────────┘
```

### 왜 TG 채널을 쓰는가 (v1에서 배제했던 이유와 반박)
- v1 주장: "Whale Alert를 쓰면 래퍼다".
- v2 반박: **공개 TG 방송을 수신하는 것은 API 키 발급·과금·ToS 가입 없이 누구나 가능한 정보 소비**이며, 게다가 우리는 그 위에 (1) Known Whale polling 교차검증, (2) 자체 시그널 규칙, (3) Claude 해석, (4) 개인화를 얹는다. 단순 번역 래퍼가 아니라 **신호의 한 축**으로 사용.
- 비용: $0. 중단 리스크: TG 봇 차단 정책 변화 시 대응 필요.

### 기술적 주의 — Bot API가 아니라 User API 필요
- python-telegram-bot(Bot API)은 **채널 멤버가 아닌 봇이 메시지를 읽을 수 없음**. `@whale_alert_io` 채널에 봇을 admin으로 초대할 수 없으므로 Bot API로는 수집 불가.
- 해결책: **Telethon**(User API, MTProto)로 개인 텔레그램 계정을 이용한 listener 구현. 계정은 수집 전용 세션 생성.
- 또는 t.me preview HTML을 파싱하는 보조 경로(지연 有). Telethon이 정석.

## 2. 감시 대상 및 시그널 (v1 재사용 + 보강)

v1 §1(감시 대상 4분류: CEX hot/cold, Bridge, Smart Money, Token Whale)과 §5(시그널 규칙 5개)는 그대로 유효. 추가로:

### 2-A. TG 이벤트 파서
`@whale_alert_io` 메시지는 대체로 정형 패턴:
> "🚨 500 #BTC (42,350,000 USD) transferred from unknown wallet to #Binance"

- 필드 추출: amount, asset, usd_value, from_category, to_category, chain(추정).
- 정규식 1차 + LLM 폴백(낮은 신뢰도일 때만, 비용 통제).
- 중복 제거: `address_activity` polling 결과와 `tg_whale_events` 교차. 동일 tx 해시 감지되면 `source = "both"`로 표시 → **교차검증된 시그널에 점수 가산**.

### 2-B. 신규 시그널 규칙 (v1 5개에 추가)
6. **tg_cex_inflow_burst**: TG 채널에서 특정 CEX로의 유입 메시지가 10분 윈도우 내 N건 초과.
7. **corroborated_move**: TG 채널 이벤트와 Known Whale polling 이벤트가 같은 tx_hash로 일치 → confidence=high.
8. **weekly_net_accumulation**: 카테고리별 7일 순유출/유입 합산 대비 직전 4주 평균의 편차.

## 3. 사용자 개인화 (자연어 기반)

### 3-A. 입력 경로
- `/start` → 기본 구독
- `/preferences` → 현재 관심사 보여주기
- **일반 메시지(명령 아님)** → Claude가 의도 추출
  - "BTC 고래만 보고 싶어" → `interests = ["BTC"], weight=1.5`
  - "솔라나 중심으로 짧게" → `chains=["solana"], style="short"`
  - "스마트머니 덜 중요해" → `categories_deweight = ["smart_money"]`

### 3-B. 저장 스키마 — `user_interests` 탭 (신규)
| 컬럼 | 설명 |
|---|---|
| chat_id | 사용자 |
| dimension | keyword / chain / category / style |
| value | BTC / solana / cex_hot / short |
| weight | 기본 1.0, 자연어 해석 결과 0.5~2.0 |
| source_message | 원문 |
| created_at / updated_at | ISO |
| active | true/false |

### 3-C. 적용 로직
- 시그널 랭킹 시 (score × ∏ weight_applicable). 가중치는 소폭(0.7~1.5)만 허용해 공통 고품질 시그널이 완전히 배제되지 않도록.
- 스타일 선호는 브리프 프롬프트에 전달("이 사용자는 짧은 요약 선호").

## 4. 산출물 포맷

### 4-A. 일간 브리프 (기존)
시그널 기반으로 top 5~7 선정 → Claude 내러티브.

### 4-B. 주간 트렌드 코멘터리 (신규)
- 매주 월요일 KST 08:00 발송.
- 내용: 카테고리별 Net Accumulation 표, 직전 4주 대비 편차, Claude의 2~3문장 해석.
- 예시 출력 형식:
  > **이번 주 요약 (4/7–4/13)**
  > - BTC: CEX 순유출 +18% vs 4주 평균 → 중기 매집 신호 약함→중간
  > - SOL: Bridge 순유입 -22% → L2로의 자금 이탈 둔화
  > - Smart Money: ARB 관련 8개 지갑 공통 순매수 → 관찰 대상

### 4-C. Streamlit 대시보드 (Phase 3 이후)
- 타임라인 카드(시그널 단위), 주간 아카이브, 사용자별 관심사 편집.

## 5. Sheets 스키마 전체 (v1 + v2 보강)

**유지**: `transactions`, `daily_brief`, `subscribers`, `analysis_log`, `system_log`

**v1 신설**: `watched_addresses`, `address_activity`, `signals`

**v2 신설**:
- `tg_whale_events`: raw_text, parsed_amount, parsed_asset, parsed_usd, parsed_from, parsed_to, tg_message_id, received_at
- `user_interests`: §3-B 스키마
- `weekly_trend`: week_start, category, net_flow_usd, prev_4w_avg, deviation_pct, commentary, created_at
- `curation_log`: date, top_signals(json), key_trend, created_at (Grok 제안의 기록 레이어)

## 6. 기술 스택 확정

| Layer | 선택 | 비고 |
|---|---|---|
| LLM | Claude 3.5 Sonnet | 요약·트렌드·자연어 해석 모두 |
| 온체인 polling | Etherscan API v2 / Solscan | §v1과 동일, Free tier |
| TG 수신 | **Telethon** (User API) | Bot API로는 불가 |
| TG 배포 | python-telegram-bot (Bot API) | 기존 유지 |
| 가격 | CoinGecko Free | 기존 유지 |
| 저장 | Google Sheets | 단기 유지, 사용자/신호 증가 시 SQLite or Supabase 마이그 |
| 스케줄 | GitHub Actions | polling 10분 주기, 일간 08:00 KST, 주간 월요일 08:00 KST |
| 대시보드 | Streamlit (비번 보호) | 기존 유지 |

## 7. 로드맵 (v1 4 Phase → v2 5 Phase)

### Phase 0 — 준비 (1일, 변동 없음)
- Etherscan/Solscan 키, Telethon용 Telegram API id/hash 발급
- `watched_addresses` 시드 80개 CSV
- `config/signals.yaml` 초안

### Phase 1 — 수집기 교체 + TG 리스너 (3~4일, v2에서 확장)
- `EtherscanCollector`, `SolscanCollector`, `registry.py` (v1과 동일)
- **신규**: `TelethonListener` 백그라운드 프로세스. 수신 메시지 파싱 → `tg_whale_events` 적재
- 신규 Sheets 탭 전부 생성
- `whale_alert.py` 제거

### Phase 2 — 시그널 엔진 + 교차검증 (3일)
- v1의 5개 + v2의 3개(tg_*, corroborated_move, weekly_net_accumulation)
- 교차검증 로직: tx_hash·시간·금액 유사도 기반 매칭
- 규칙별 unit test

### Phase 3 — 개인화 루프 (2일, v2 신설)
- Claude 프롬프트: 사용자 자유 메시지 → `user_interests` diff
- 시그널 랭킹에 weight 적용
- `/preferences` 핸들러

### Phase 4 — 산출물 & 배포 (2일)
- 일간 브리프 (시그널+weight 기반)
- 주간 트렌드 코멘터리 (신규 cron)
- Streamlit 대시보드 확장

### Phase 5 — 품질 루프 (지속)
- `signal_outcome` 트래킹: 각 시그널 이후 24h 가격 반응 기록
- 주 1회 precision 리뷰
- 감시 주소 리스트 월간 보강
- 개인화 품질: 사용자 피드백 명령("이건 별로") 반영

## 8. 비용 · 리스크 업데이트

| 리스크 | 영향 | 완화 |
|---|---|---|
| TG 수신 계정 차단/밴 | TG 소스 실종 | 전용 계정 사용, rate 준수, Etherscan polling으로 부분 복구 |
| Telethon 세션 만료 | 장애 | 세션 재인증 스크립트 + Actions 실패 알림 |
| Etherscan Free 한도 초과 | 수집 중단 | 주소·주기 조정, 200 주소/15분까지는 여유 (§v1 §3) |
| TG 메시지 포맷 변경 | 파싱 실패 | 정규식 + Claude 폴백 이중 구조 |
| 자연어 개인화 오해석 | UX 저하 | weight 범위 0.7~1.5로 제한, `/preferences`로 사용자 확인 |
| 교차검증 매칭 오류 | false corroboration | 매칭은 tx_hash 우선, 미해시 이벤트는 "single-source" 태그 고정 |

**총 예상 비용**: 여전히 $0/월. Claude API 사용만 변수(주간 코멘터리 + 자연어 해석 추가로 기존 대비 ~30% 증가 예상, Top5 캐시 활용하면 상쇄).

## 9. 차별화 포인트 재정의

기존(v1): "자체 수집 + 시그널 + AI 해석"
v2 추가:
- **하이브리드 소스 교차검증** → 시그널 신뢰도를 공식화 가능 (confidence tier)
- **자연어 개인화** → 명령형 UI가 아닌 대화형 UX (Grok 제안)
- **주간 트렌드 코멘터리** → 일회성 알림을 넘어 **투자자가 월요일 아침에 읽고 싶어하는 포맷**
- **커뮤니티 확장성**: `user_interests` 집계로 "이번 주 가장 많이 언급된 코인" 같은 메타 콘텐츠 생성 가능(향후)

## 10. 성공 기준

- 기능: `grep -ri "whale.alert.*api" src/` 결과 0건. TG는 `telethon.*@whale_alert_io`만.
- 비용: 월 $0 (Claude 사용료 제외).
- 품질: 시그널 precision 60%+ (Phase 5 기준), 교차검증된 시그널은 70%+ 목표.
- 개인화: 자연어 해석 정확도 80%+ (샘플 50건 수동 라벨링).
- 산출물: 일간 브리프 + 주간 코멘터리 모두 자동 발송.
