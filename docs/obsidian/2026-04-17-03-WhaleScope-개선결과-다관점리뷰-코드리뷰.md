## 2. 다관점 리뷰

### 2.1 Engineering 관점

**강점**
- 커밋이 **논리 단위로 분리**되어 있어 리뷰·롤백 비용이 낮다. `feat/refactor/fix/test/docs/chore` 태그가 일관되게 적용됨.
- `asyncio.to_thread`를 **블로킹 I/O 3개소 전부**에 정확히 넣어 이벤트 루프 정지를 차단함 (`router.call_task`, 2× `storage.*`).
- `health_status()`가 900초 임계치로 stale 판정을 하며, heartbeat 300초 간격과 `3×` 간격의 관계가 **의도적**으로 설계됨 (한 번 놓쳐도 즉시 알람으로 직결되지 않음).
- `list_tg_whale_events` 경계 테스트 6개는 **TDD 품질 기준**에 부합 — tz-naive × tz-aware 매칭, 빈 시트, 미래 since, limit 0·None 의미까지 모두 커버.
- `Storage` protocol 타입을 `list[dict[str, object]]`로 좁혀 duck-typing 계약을 명문화 — 이후 SheetsClient 외 구현체(테스트용 InMemoryStorage 등)를 만들 때 타입 체커가 계약 위반을 잡아줌.

**약점/위험**
- **SystemLogPanel 고아 상태 (H0, 중대)**: `apps/dashboard/components/system-log-panel.tsx` 전체 132줄이 작성됐으나, grep 결과 `SystemLogPanel`을 import하는 곳이 **존재하지 않음** (`page.tsx`는 자체 `oplog-card` / `oplog-list` 인라인 마크업 사용). 모달 UX가 실제 화면에 노출되지 않아 데모 시 보이지 않음. CSS 셀렉터(`.log-modal__*`)는 `globals.css`에 살아있어 dead code 동기화 부담만 늘어남.
- **`_safe_float` 중복 정의 (M1, 중간)**:
  - `src/main.py:113` — `float(str(value).replace(",", ""))`, 실패 시 `logger.warning`
  - `src/signals/baseline.py:18` — `float(value or 0.0)`, `field_name` 인자, 실패 시 `logger.debug`
  - **시맨틱이 다름**: main 버전은 `"1,234.5"` 같은 comma-number를 파싱하지만 baseline 버전은 불가. 한쪽이 다른 쪽으로 대체되면 조용한 회귀가 발생한다. 공용 유틸(`src/utils/number_utils.py`)로 흡수하면서 comma 처리 옵션을 인자로 명시하는 방향 권장.
- **`SheetsClient` 락 부재 (M2, 중간)**: `src/storage/`에 `Lock`/`asyncio.Lock` 정의 0건. 리스너는 `asyncio.to_thread`를 써서 여러 append를 동시 schedule할 수 있고, 파이프라인도 async 컨텍스트에서 append를 부를 수 있음. gspread의 worksheet 핸들은 스레드-세이프하다고 보장되지 않으므로 이론적 레이스가 존재. 최소한 **write 계열 메서드에 `threading.Lock`을 래핑**하는 쪽이 안전. (gspread 자체가 HTTP 호출이라 Lock은 throughput 손해가 크지 않음.)
- **`_tg_direction`의 `cex_to_cex` 다운스트림 정합**: `signals/rules.py`의 모든 CEX 룰이 `counterparty_category == "cex"` 동등 비교를 하므로 `cex_to_cex`는 **의도적으로 매칭되지 않음** — 이 설계는 노이즈 제거라는 원래 목적에 부합한다(✓). 다만 **해당 거동이 어디서도 문서화되지 않음** — `rules.py` 상단 docstring이나 `config/signals.yaml` 주석에 "CEX 룰은 `cex_to_cex`를 고의로 제외한다"를 기록해 두어야 장차 룰 확장 시 실수로 `in ("cex", "cex_to_cex")`로 고쳐 의미가 역전되는 것을 막는다.
- **`list_tg_whale_events` limit 의미 (L1, 낮음)**: `rows[-limit:] if limit else []`는 `limit == 0` → 빈 리스트, `limit is None` → 전체인데, 이 분기 의미가 호출자 입장에서 명확하지 않음. protocol / 구현 docstring에 "limit=0 means none, limit=None means all"을 명시 권장.

**기타**
- `parse_dt` 공용 유틸화(A1)는 테스트 전반에 파급 없이 안정 반영. `main.py`/`baseline.py`의 `from src.utils.datetime_utils import parse_dt as _parse_dt  # noqa: E302` 스타일 주석이 반복되는데, 한 곳에서 alias를 없애고 `parse_dt`를 직접 쓰는 쪽이 더 깔끔함 — 소규모 청소 후보.

### 2.2 PM / Investor 관점

**강점**
- **평가 임팩트 관점에서 ‘관찰 가능성(observability)’ 스토리가 완성**됨: (i) 리스너 health_status, (ii) 시스템 로그 human-readable, (iii) 서비스 카드에서 원클릭 앵커 이동. 이는 과제 평가 루브릭에서 "운영 가능성"/"실제 서비스 수준" 항목에 직접 득점되는 영역.
- Gemini 2.5 단가 추가와 `status` 필드 추가로 **비용/실패율 모니터링**의 토대가 마련됨. 투자자/심사관 데모에서 "모델 비용을 의식하며 설계했다" 메시지로 전환 가능.
- `brief_texts` 파이프라인이 완결돼 **1인 1텍스트** 개인화 경로가 실질적으로 동작. 다만 다운스트림(실제 LLM이 signal_id별로 텍스트를 생성하는 코드 경로)이 연결돼 있는지 별도 점검 필요(다음 스프린트).

**위험/고민**
- **B4의 부분 이행이 데모 화면에 직접적으로 보인다**: 계획서에서 "로그 클릭 → 모달 상세"는 심사관에게 보여주기 좋은 인터랙션인데, 현재 메인 대시보드에서는 로그를 클릭해도 아무 반응이 없다. 데모 시연 전에 **반드시** page.tsx에 SystemLogPanel을 연결하거나, 최소한 기존 `oplog-item`에 동일한 모달 훅을 붙여야 함.
- **"왜 이걸 고쳤는가"의 서사 부족**: README는 아키텍처를 기술하지만, 이번 스프린트의 개선 사유(예: "리스너 블로킹이 Telegram polling을 막던 문제를 `asyncio.to_thread`로 해소")를 1페이지 changelog 형태로 정리하면 평가 시 유리하다. `docs/changelog.md` 혹은 `docs/engineering-notes.md` 신규 작성 권장.
- **C-H2 (운영 임계치 .env 이관)**: 계획서 C-H2 항목의 진척 여부는 확인되지 않음. 900초가 여전히 하드코딩이라면 데모 환경(저부하)에서 쉽게 stale 경보가 뜰 수 있어 민감도 조정 여지가 없음.

### 3.2 중간 (Medium)

**M1. `_safe_float` 중복 정의 — 시맨틱 불일치**
- `src/main.py:113-118`
  ```python
  def _safe_float(value: object) -> float:
      try:
          return float(str(value).replace(",", ""))
      except (TypeError, ValueError):
          logger.warning("Failed to parse float value=%r; defaulting to 0.0", value)
          return 0.0
  ```
- `src/signals/baseline.py:18-27`
  ```python
  def _safe_float(value: object, field_name: str | None = None) -> float:
      try:
          return float(value or 0.0)
      except (TypeError, ValueError):
          logger.debug("_safe_float failed field=%s value=%r; defaulting to 0.0", field_name, value)
          return 0.0
  ```
- **권장 조치**:
  ```python
  # src/utils/number_utils.py (신규)
  def safe_float(
      value: object,
      *,
      field_name: str | None = None,
      strip_commas: bool = False,
      log_level: int = logging.DEBUG,
  ) -> float:
      ...
  ```
  양쪽 호출자를 명시적 옵션으로 이관. 시맨틱 차이를 옵션으로 드러내야 실수 방지.

**M2. `SheetsClient`에 async-safe 락 없음**
- `src/storage/sheets_client.py` 전반 — `append_*` 메서드들이 `worksheet.append_row` 등을 직접 호출.
- 호출자: `src/ingestion/telethon_listener.py:226,257` (`asyncio.to_thread`), 기타 파이프라인 async 구간.
- **권장 조치**: 클래스 생성 시 `self._write_lock = threading.Lock()` 보관, 모든 write 메서드에서 `with self._write_lock:` 블록으로 쓰기 경로 직렬화. 읽기 경로는 그대로 두어 성능 영향을 최소화.

**M3. `cex_to_cex` 분리 의도의 명시화**
- `src/signals/rules.py:44, 91, 252` — 모든 CEX 룰이 `counterparty_category == "cex"` 동등 비교.
- `src/main.py:137-153` — `cex_to_cex`를 별도 카테고리로 방출.
- 현재는 의도대로 필터링되지만, 주석/문서화가 없어 미래의 "확장" 수정이 역설적으로 회귀를 만들 위험.
- **권장 조치**: `src/signals/rules.py` 모듈 상단 docstring과 `config/signals.yaml` 각 CEX 룰에 `# cex_to_cex events are intentionally excluded (see src/main.py::_tg_direction)` 주석 추가.

### 3.3 낮음 (Low)

**L1. `list_tg_whale_events(limit)` 의미 문서화 부족**
- `src/storage/protocol.py` — `limit: int | None = None` 타입 표기만 있음.
- 구현 거동: `limit is None` → 전체, `limit == 0` → `[]`, `limit > 0` → 뒤에서 `limit` 개.
- **권장 조치**: 프로토콜과 구현의 docstring에 위 의미를 명시. 테스트는 이미 `test_respects_limit`에서 유효값을 커버하지만, `limit == 0`의 의미는 호출자가 알기 어려움.

**L2. `_parse_dt` alias의 불필요**
- `src/main.py:110`, `src/signals/baseline.py:13`에서 `from src.utils.datetime_utils import parse_dt as _parse_dt  # noqa: E302`
- 실제로 `_parse_dt`로 쓰이지만 이름을 바꿀 필요가 없음.
- **권장 조치**: `parse_dt` 그대로 import하고 호출부에서 `parse_dt(...)`로 사용. 작은 청소.

**L3. `health_status` staleness 임계치의 .env 이관(계획 C-H2 잔여 여부)**
- `src/ingestion/telethon_listener.py` 내 `900` 매직넘버가 남아있다면 `LISTENER_STALENESS_SECONDS` 환경변수로 이관 권장 (아직 미확인, 확인 필요).

### 3.4 Positives (유지·강화할 부분)

- `timingSafePasswordEquals`의 **length pre-check**: Node `crypto.timingSafeEqual`이 길이 불일치 시 throw하는 문제를 가드로 차단. 보안 상세를 이해한 구현.

- `test_to_thread_offloads_router_and_storage`: **설계 계약(루프 언블록)을 테스트로 표현**한 좋은 사례.
- 커밋 메시지 스타일 일관성 (`type(scope): summary`). `feat/fix/refactor/chore/test/docs` prefix 일관 적용.
