## Task 1. 브리핑 hero summary 축약

### 문제

`app/page.tsx:638`의 `<p className={styles.heroSummary}>&ldquo;{brief.summary}&rdquo;</p>`가 줄바꿈 제어 없이 LLM 생성 요약(300~500자)을 그대로 렌더. 모바일/태블릿에서 hero 영역을 넘치게 채움.

### 구현

- `lib/format.ts`에 `truncateBriefHeadline(value, maxChars = 110)` 추가.
  - 1차: 맨 앞 110자 안에 문장 경계(`.!?。！？`)가 있으면 그 지점에서 절단 (최소 55자 이상일 때).
  - 2차: 경계 없으면 마지막 공백에서 절단하여 `…` 부가.
- `app/page.tsx`에서 hero p 태그만 `truncateBriefHeadline(brief.summary)` 사용. `brief.summary` 전체는 analysis 항목(핵심 해석)에서 계속 노출.

### 변경 파일

- `apps/dashboard/lib/format.ts` (+12)
- `apps/dashboard/app/page.tsx` (+1 -1 + import 1줄)

### 커밋

### 3-1. Phase A — 홈 직렬 await 병렬화 + admin extras 스킵

**진단**:
- `app/page.tsx:478` `InsightsPage`가 dictionary→language→state→fearGreed를 순차 `await`. 가장 느린 `loadInsightState`가 전체를 블록.
- `loadInsightState` → `getDashboardData()`는 내부에서 `loadRenderObservability()`(Render REST, cold 시 2~3s) + 4개 optional 시트(`brief_cost_ledger`, `broadcast_log`, `llm_budget_log`, `watched_addresses`) + `buildAdminObservabilitySummary`까지 호출. 홈은 이 중 `adminObservability`를 쓰지 않음.

**적용**:

```ts
// app/page.tsx:478
export default async function InsightsPage() {
  const [dictionary, language, state, fearGreed] = await Promise.all([
    getCurrentDashboardDictionary(),
    getCurrentDashboardLanguage(),
    loadInsightState(),
    getFearGreedData(),
  ]);
  ...
}

// app/page.tsx:455 loadInsightState
const data = await getDashboardData({
  transactionLimit: 6,
  signalLimit: 6,
  systemLogLimit: 4,
  includeAdminExtras: false,   // ← 추가
});
```

```ts
// lib/metrics.ts:2911 getDashboardData
export async function getDashboardData(options?: {
  transactionLimit?: number;
  signalLimit?: number;
  systemLogLimit?: number;
  includeAdminExtras?: boolean;   // ← 추가 (기본 true, 기존 호출자 호환)
}): Promise<DashboardData> {
  ...
  const includeAdminExtras = options?.includeAdminExtras ?? true;
  // Promise.all 슬롯 6개를 조건부 Promise.resolve([])로 치환
  // buildAdminObservabilitySummary도 조건부 호출 → null 할당
}
```

`/admin`(Admin 페이지)은 옵션 미지정 → 기존과 동일한 full payload 수신.
