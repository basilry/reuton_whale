# WhaleScope 다관점 리뷰 · 개선안 실행 계획

> **기준 문서**: [[2026-04-22-01-WhaleScope-다관점리뷰-종합보고서]] (6.84/10, Security Critical 1건)
> **작성 목적**: 종합 보고서에서 도출된 20개 액션을 **실제로 실행 가능한 작업 단위**로 분해. 각 액션에 "현황 → 변경 코드 → 검증 → 롤백 → 소요 → 의존성" 6필드를 명시.
> **원칙**: (1) Phase 0은 **2~3시간 내 완료 · 커밋 가능** 단위로 쪼갠다. (2) Phase 1 이후는 "PR 한 건 = 1 액션" 단위로 설계한다. (3) 각 액션은 **롤백 가능**해야 한다 — 단일 env flag 또는 단일 파일 revert로 이전 상태로 돌릴 수 있어야 한다.

## 0. 본 문서의 사용법

이 문서는 **체크리스트 + 실행 매뉴얼**이다. 상단부터 순서대로 읽을 필요는 없다. 다음 3가지 방식 중 하나를 택해 쓴다.

1. **제출 마감 전 2~3시간 남았다 → §3 Phase 0만 본다.** 6개 액션이 독립적이므로 병렬 진행 가능.
2. **제출 직후 1주 스프린트 계획 → §4 Phase 1을 본다.** 7개 액션이 `[P1-1] PR-gated CI → [P1-2~7]` 순으로 의존성이 있다.
3. **입사 후 1개월+ 로드맵 → §5 Phase 2 ~ §6 Phase 3을 본다.** 대형 파일 분할은 §7 의존성 그래프부터 먼저 확인.

각 액션은 동일한 6-필드 구조로 통일했다:

- **🎯 목표**: 이 액션이 해결하는 구체적 문제
- **📂 현황**: 정확한 파일 경로 + 라인 + 문제의 증거 코드
- **🔧 변경**: before → after 코드 diff (실행 가능 수준)
- **✅ 검증**: 변경 후 돌려야 할 커맨드/테스트
- **🔙 롤백**: 문제 발생 시 원복 방법 (env 또는 revert SHA)
- **⏱ 소요 & 🔗 의존성**: 단위는 시간, 의존은 다른 액션 번호

## 1. 실행 원칙 (CLAUDE.md 전역 규칙과 일치)

1. **이해 먼저, 수정은 나중에.** 각 액션 실행 전 `📂 현황` 블록의 파일을 반드시 Read/Grep으로 최신 상태 확인한다. 본 문서는 2026-04-22 기준이다.

6. **문서화와 동시 진행.** 각 액션 완료 시 `docs/changelog.md`에 Why/What/How validated/Risks 4-필드로 기록. v6+ 백필은 Phase 3 [P3-4]에서 한 번에.

### [P0-2] `/api/debug/dashboard` 프로덕션 차단 + `timingSafeEqual` (Security Critical)

**🎯 목표**
- `mode=env`/`mode=redisPing`의 **무인증 정찰(recon) 경로** 차단.
- 토큰 비교 `provided === expected`를 timing-safe 비교로 교체.

**📂 현황** — `apps/dashboard/app/api/debug/dashboard/route.ts:14-45`
```ts
function tokenMatches(request: Request): boolean {
  const url = new URL(request.url);
  const provided = url.searchParams.get("token")?.trim();
  const expected = process.env.DASHBOARD_PASSWORD?.trim();
  if (!expected || !provided) return false;
  return provided === expected; // ⚠️ non-constant-time
}
// ...
const publicModes = new Set(["env", "redisPing"]);
if (!publicModes.has(mode) && !tokenMatches(request)) {  // ⚠️ env/redisPing 완전 공개
  return NextResponse.json({ error: "forbidden" }, { status: 403 });
}
```
- `mode=env` 응답(57-87행)은 `GOOGLE_SHEET_ID`, `GOOGLE_CREDENTIALS_JSON`, `RENDER_API_KEY`, `TELEGRAM_BOT_TOKEN` 등 **9개 이상 민감 env 키의 존재 여부**를 bool로 반환. 공격자가 서비스 스택을 역으로 매핑할 수 있음.
- `mode=redisPing` 응답(89-107행)은 Upstash Redis 엔드포인트 `/ping` 결과 **raw body 200자**를 포함. Upstash 주소·응답 형태 노출.

**🔧 변경**
```ts
// top of file
import { timingSafeEqual } from "node:crypto";

// before: tokenMatches
function tokenMatches(request: Request): boolean {
  // ...
  return provided === expected;
}

// after: tokenMatches
function tokenMatches(request: Request): boolean {
  const url = new URL(request.url);
  const provided = url.searchParams.get("token")?.trim();
  const expected = process.env.DASHBOARD_PASSWORD?.trim();
  if (!expected || !provided) return false;

  const a = Buffer.from(provided, "utf8");
  const b = Buffer.from(expected, "utf8");
  if (a.length !== b.length) return false;
  return timingSafeEqual(a, b);
}

// before: GET handler 상단
const publicModes = new Set(["env", "redisPing"]);
if (!publicModes.has(mode) && !tokenMatches(request)) {
  return NextResponse.json({ error: "forbidden" }, { status: 403 });
}

// after: GET handler 상단
if (process.env.NODE_ENV === "production") {
  // Production에서는 모든 debug endpoint를 완전히 닫음(404).
  // env/redisPing도 예외 없음 — 운영 관측은 /api/admin/health 경유.
  return new NextResponse("Not Found", { status: 404 });
}
if (!tokenMatches(request)) {
  return NextResponse.json({ error: "forbidden" }, { status: 403 });
}
```

# before
# KST 기준 full 브리핑 실행 슬롯 (09/15/21시)
_FULL_BRIEF_HOURS_KST = frozenset({9, 15, 21})

# after
import json as _json
from pathlib import Path as _Path

def _load_brief_schedule() -> frozenset[int]:
    path = _Path(__file__).resolve().parent.parent.parent / "config" / "brief_schedule.json"
    try:
        with path.open("r", encoding="utf-8") as fp:
            data = _json.load(fp)
        hours = data.get("fullBriefHoursKst")
        if not isinstance(hours, list) or not all(isinstance(h, int) and 0 <= h <= 23 for h in hours):
            raise ValueError(f"invalid fullBriefHoursKst in {path}: {hours!r}")
        return frozenset(hours)
    except FileNotFoundError:

**Step 3**: TypeScript 측 로더 — `apps/dashboard/lib/brief-schedule.ts` 신규
```ts
import { readFileSync } from "node:fs";
import { resolve } from "node:path";

type BriefSchedule = { fullBriefHoursKst: number[] };

let cached: readonly number[] | undefined;

export function getBriefScheduleHoursKst(): readonly number[] {
  if (cached) return cached;
  try {
    const path = resolve(process.cwd(), "..", "..", "config", "brief_schedule.json");
    const raw = readFileSync(path, "utf8");
    const parsed = JSON.parse(raw) as BriefSchedule;
    if (
      !Array.isArray(parsed.fullBriefHoursKst) ||
      !parsed.fullBriefHoursKst.every((h) => Number.isInteger(h) && h >= 0 && h <= 23)
    ) {
      throw new Error("invalid brief_schedule.json shape");
    }
    cached = [...parsed.fullBriefHoursKst].sort((a, b) => a - b);
    return cached;
  } catch (error) {
    console.warn("[brief-schedule] fallback to default", error);
    cached = [9, 15, 21];
    return cached;
  }
}
```

**Step 1**: `scripts/snapshot_telegram_metrics.py` 신규 스크립트
```python
"""
Telegram 채널 메트릭 스냅샷.
Usage: python scripts/snapshot_telegram_metrics.py > docs/metrics/tg_snapshot_$(date +%Y%m%d).md
"""
import os, json, datetime, urllib.request

BOT_TOKEN = os.environ["TELEGRAM_BOT_TOKEN"]
CHANNEL = os.environ.get("TELEGRAM_CHANNEL_USERNAME") or os.environ["TELEGRAM_CHANNEL_ID"]

def tg(method: str, **params):
    url = f"https://api.telegram.org/bot{BOT_TOKEN}/{method}"
    data = json.dumps(params).encode()
    req = urllib.request.Request(url, data=data, headers={"content-type": "application/json"})
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read())["result"]

count = tg("getChatMemberCount", chat_id=CHANNEL)
chat = tg("getChat", chat_id=CHANNEL)
now = datetime.datetime.now(datetime.timezone.utc).astimezone().isoformat(timespec="seconds")

print(f"""# Telegram 채널 스냅샷 — {now}

| 지표 | 값 |
|------|----:|
| 채널 | `{CHANNEL}` |
| 구독자 수 | **{count}** |
| 채널 타이틀 | {chat.get("title", "—")} |
| 스냅샷 시각 | {now} |

> 발송 건수·view count는 `tg_whale_events` 시트의 최근 N건 집계로 보완.
""")
```

const CSP_DIRECTIVES = {
  "default-src": ["'self'"],
  "script-src": ["'self'", "'unsafe-inline'", "https://va.vercel-scripts.com"],
  "style-src": ["'self'", "'unsafe-inline'", "https://fonts.googleapis.com"],

  "object-src": ["'none'"],
  "base-uri": ["'self'"],
  "form-action": ["'self'"],
};

function cspString(): string {
  return Object.entries(CSP_DIRECTIVES)
    .map(([k, v]) => `${k} ${v.join(" ")}`)
    .join("; ");

### [P1-3] Upstash Redis 기반 분산 rate-limit (Security + Performance)

**🎯 목표**
- 종합보고서 §4 Top 3 Security #3: `rate-limit.ts`의 in-memory `buckets: Map<string, Bucket>`은 서버 인스턴스가 2개 이상이면 bypass 가능. Render 무료 인스턴스도 재시작 시 리셋.
- `clientKey()`가 `x-forwarded-for` 첫 값만 신뢰 → 공격자 헤더 조작 가능.

**📂 현황** — `apps/dashboard/lib/rate-limit.ts:14-41`, `:62-69`

**🔧 변경**
```ts
// apps/dashboard/lib/rate-limit.ts 교체
import { NextResponse } from "next/server";
import { Redis } from "@upstash/redis";

// 전용 REST URL/Token 사용. WHALESCOPE_REDIS_* 와 동일한 인스턴스.
const redis =
  process.env.WHALESCOPE_REDIS_REST_URL && process.env.WHALESCOPE_REDIS_REST_TOKEN
    ? new Redis({
        url: process.env.WHALESCOPE_REDIS_REST_URL,
        token: process.env.WHALESCOPE_REDIS_REST_TOKEN,
      })
    : null;

export const API_RATE_LIMIT = { windowMs: 60_000, maxRequests: 30 };

export async function rateLimit(
  key: string,
  config = API_RATE_LIMIT,
): Promise<{ allowed: boolean; retryAfter?: number }> {
  // 분산 환경: Upstash INCR + EXPIRE (sliding window approximation)
  if (redis) {
    const bucket = `rl:${key}:${Math.floor(Date.now() / config.windowMs)}`;
    const pipe = redis.pipeline();
    pipe.incr(bucket);
    pipe.expire(bucket, Math.ceil(config.windowMs / 1000) + 1);
    const [count] = (await pipe.exec()) as [number, unknown];
    if (count > config.maxRequests) {
      const retryAfter = Math.ceil(config.windowMs / 1000);
      return { allowed: false, retryAfter };
    }
    return { allowed: true };
  }

  // 로컬 dev 폴백: 기존 in-memory.
  return rateLimitLocal(key, config);
}

// clientKey: trusted forwarded chain 검사
export function clientKey(request: Request): string {
  const forwarded = request.headers.get("x-forwarded-for");
  if (forwarded) {
    // Render/Vercel은 마지막 IP가 실제 클라이언트. 첫 IP는 조작 가능.
    const parts = forwarded.split(",").map((s) => s.trim()).filter(Boolean);
    const client = parts[parts.length - 1];
    if (client) return client;
  }
  return request.headers.get("x-real-ip") || "unknown";
}

// rateLimitLocal (기존 in-memory 구현을 함수로 분리, dev 환경 전용)
```

private valuesToRows<T extends SheetTabName>(
  tab: T,
  values: string[][] | undefined,
): SheetRowMap[T][] {
  if (!values || values.length <= 1) return [];
  const schema = TAB_SCHEMAS[tab];
  const headers = TAB_HEADERS[tab];
  const rows: SheetRowMap[T][] = [];
  for (const rawRow of values.slice(1)) {
    const obj = Object.fromEntries(headers.map((h, i) => [h, rawRow[i] ?? ""]));
    const result = schema.safeParse(obj);
    if (!result.success) {
      console.warn(`[sheets] drift ${tab}`, result.error.issues.slice(0, 3));
      continue; // drop malformed row, don't throw
    }
    rows.push(result.data as SheetRowMap[T]);
  }
  return rows;
}

**변경 설계**
```python
# src/pipeline/brief_stages.py 신규
from typing import Protocol

class BriefStage(Protocol):
    name: str
    async def run(self, ctx: "BriefContext") -> None: ...

class CollectSignalsStage:
    name = "collect_signals"
    async def run(self, ctx): ...

class AssembleBriefStage: ...
class CallLLMStage: ...
class PublishStage: ...

# src/pipeline/brief.py
async def run_brief_pipeline(*, mode: BriefMode) -> BriefResult:
    ctx = BriefContext.bootstrap(mode=mode)
    stages: list[BriefStage] = [
        CollectSignalsStage(),
        AssembleBriefStage(),
        CallLLMStage(),
        PublishStage(),
    ]
    try:
        for stage in stages:
            async with _stage_timer(stage.name, ctx):
                await stage.run(ctx)
        return ctx.result()
    finally:
        await finalize_brief_run(ctx)  # heartbeat·cost_ledger·cleanup
```
