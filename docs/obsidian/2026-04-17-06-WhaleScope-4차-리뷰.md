## 2. Code Review — 신규 이슈

### NB1. `page.tsx` God Component (High)

`apps/dashboard/app/page.tsx`가 **1,220 LOC** 단일 파일. 타입 정의(12종), 정규화 함수(6종), 한국어 humanize 함수(15종), 렌더 로직이 전부 한 곳에 있다.

**영향**: 유지보수·테스트 곤란. 렌더링 로직 변경 시 타입/유틸 실수 유발. 리뷰어가 변경 범위를 판단하기 어려움.

**조치**: 3개 모듈로 분리.
- `types.ts` — DashboardMetrics, NormalizedBrief 등 12종
- `humanize.ts` — humanizeSignal, humanizeTransaction, ruleLabel 등 15개 함수 (현재 `lib/format.ts`에 일부 존재하므로 통합)
- `normalize.ts` — normalizeDashboardData, normalizeTransactions, normalizeSignals

### NB2. `_signal_to_top_item` falsy-zero 버그 (Medium)

`main.py:247`:
```python
amount_usd: float | None = sum(e.amount_usd for e in deduped_events) or None
```
`sum()`이 `0.0`을 반환하면 `0.0 or None`은 `None`으로 평가. 모든 이벤트의 amount_usd가 0인 정당한 케이스(미지원 심볼)에서 amount가 "unknown"으로 보고됨.

**조치**: `amount_usd = sum(...); amount_usd = amount_usd if amount_usd > 0 else None` 또는 명시적 `total = sum(...); amount_usd = total if total else None` (0.0을 None으로 취급하는 의도라면 주석 추가).

### NB3. `main.py` 696 LOC 비대 (Medium)

3차 리뷰 시점(391 LOC)에서 78% 증가. 순수 파이프라인 오케스트레이션 외에 TG 이벤트 파싱(`_tg_row_to_event`, `_normalize_tg_chain`, `_tg_direction`), 시그널 변환(`_signal_to_top_item`, `_signal_to_sheet_dict`), 이벤트 직렬화(`_event_to_address_activity`) 등 독립 모듈로 분리 가능한 헬퍼가 13개.

**조치**: `src/pipeline/converters.py` 또는 `src/ingestion/tg_normalizer.py`로 TG 관련 5개 함수 추출. `src/signals/formatters.py`로 signal→dict 변환 2개 추출.

### NB4. Dashboard API 캐싱 부재 (Medium)

`apps/dashboard/app/api/dashboard/route.ts`는 `force-dynamic`으로 매 요청마다 Google Sheets API 호출. 동시 접속 시 gspread 레이트 리밋 즉시 도달.

**조치**: Next.js ISR (`revalidate: 60`) 또는 응답 캐시 미들웨어 추가. 대시보드 데이터는 1분 TTL로 충분.

### NB5. `formatUsd(0)` 버그 (Medium)

`page.tsx:202-203`:
```tsx
function formatUsd(value?: number) {
  if (!value) { return "USD 환산값 없음"; }
```
`value=0`일 때 `!0 === true`이므로 0 USD를 "환산값 없음"으로 표시. 실제로는 금액이 0인 거래(수수료만 있는 경우 등)도 존재.

**조치**: `if (value == null || value === undefined)` 또는 `if (typeof value !== 'number')`.

### NB6. pytest.ini에 `asyncio_mode` 미설정 (Low)

`pytest-asyncio`가 requirements.txt에 있지만 `pytest.ini`에 `asyncio_mode = auto` 미설정. 결과적으로 모든 async 테스트에 `@pytest.mark.asyncio` 데코레이터가 필요하고, 설치 누락 시 36건 일괄 실패로 원인 파악이 지연됨.

**조치**: `pytest.ini`에 `asyncio_mode = auto` 추가 + pyproject.toml 기반 의존성 관리 도입.

### NB7. `import logging` 위치 (Low)

`main.py:110` — stdlib `import logging`이 `from src...` 블록 사이에 삽입. PEP 8 위반(stdlib → third-party → local 순서).

**조치**: 파일 상단 stdlib 블록으로 이동.

### NB8. baseline.py 변수 섀도잉 (Low)

Line 61 `day = block_time.date().isoformat()` → Line 77 `_stats(daily[day]["in"] for day in chain_dates)`. 제너레이터 내 `day`가 외부 `day`를 섀도잉. 동작은 정상이나 가독성 혼란.

**조치**: 외부 변수명을 `date_key`로 변경.
