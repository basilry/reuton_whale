# WhaleScope 자체 수집 전환 계획 — Whale Alert 완전 제거

## 0. 결정 사항

- Whale Alert 의존 **완전 제거**. 현재 쓰는 Developer API는 이미 deprecated이고, 유료 전환 없이는 실운영 불가.
- 대체 구조: **공개 라벨 + 블록체인 익스플로러 API**로 고래 지갑을 직접 감시하고, 자체 휴리스틱으로 시그널을 생성한다.
- WhaleScope의 자산은 이제 "API를 번역하는 래퍼"가 아니라 **"선별된 감시 지갑 리스트 + 시그널 규칙 + AI 해석"** 세 가지.

## 1. 감시 대상 분류 (Watched Entities)

고래라는 하나의 버킷으로 묶지 않고 의미 있는 4개 카테고리로 분리한다.

### 1-A. CEX Hot/Cold Wallets — 거래소 지갑
- **왜 보나**: 대형 입출금이 가격 변동의 선행 지표. 특히 cold→hot 이동은 매도 준비로, hot→cold는 흡수로 해석 가능.
- **대표 주소 (Ethereum, 검증된 공개 라벨)**:
  - Binance 7: `0xbe0eb53f46cd790cd13851d5eff43d12404d33e8` (1.99M ETH)
  - Binance Hot Wallet 20: `0xf977814e90da44bfa03b6295a0616a897441acec` (285K ETH)
  - Binance Peg: `0x47ac0fb4f2d84898e4d9e7b4dab3c24507a6d503` (554K ETH)
  - Gemini Cold: `0xafcd96e580138cfa2332c632e66308eacd45c5da` (490K ETH)
  - Bitfinex 19/20: `0xe92d1a43df510f82c66382592a047d288f85226f`, `0x8103683202aa8da10536036edef04cdd865c225e`
  - Kraken Cold 4/3: `0x9f1799fb47b1514f453bcebbc37ecfe883756e83`, `0x8d05d9924fe935bd533a844271a1b2078eae6fcf`
  - Robinhood: `0x40b38765696e3d5d8d9d834d8aad4bb6e418e489`, `0x73af3bcf944a6559933396c1577b257e2054d935`
  - Upbit 41: `0x0e58e8993100f1cbe45376c410f97f4893d9bfcd`
  - Coinbase Prime, OKX, Bybit, Bitstamp는 별도 seed 필요
- **목표 커버리지**: ETH 기준 상위 30개 CEX 지갑 + 토큰별(ERC20) 상위 10개 입금 주소 (USDT/USDC 포함)

### 1-B. Bridge / L2 Portals — 체인 간 이동
- **왜 보나**: 대규모 브릿지 입금은 자산 이동 방향을 읽게 해준다.
- **대표 주소**:
  - Arbitrum Bridge: `0x8315177ab297ba92a06054ce80a67ed4dbd7ed3a` (818K ETH)
  - Base Portal: `0x49048044d57e1c92a77f79988d21fa8faf74e97e` (829K ETH)
  - Optimism Portal, Polygon Bridge, zkSync Bridge 추가 필요
- **모니터 대상**: 6시간/24시간 누적 순유입

### 1-C. Smart Money — 알려진 개인·펀드 지갑
- **왜 보나**: 스마트머니 1차 도입이 2차 랠리의 선행. Nansen이 팔고 있는 핵심 상품이지만 공개 라벨로도 일부 가능.
- **시드 소스**:
  - Vitalik 등 공개 인물: `Vb 3` (`0x220866b1a2219f40e72f5c628b65d54268ca3a9d`) 외
  - Arkham 공개 프로필(무료 계정으로 엔티티 단위 주소 수집 가능; 자동 스크래핑 ToS 주의 → 수동 큐레이션 원칙)
  - Dune 공개 대시보드 "Smart Money"류 쿼리 복제
  - GitHub 공개 저장소: `brianleect/etherscan-labels` (총 45K 라벨, 2023 snapshot이라 신선도는 보조)
- **초기 목표**: 수동 큐레이션 20~50개 지갑. 시그널 품질 검증 후 확장.

### 1-D. Token Whales — 특정 토큰 상위 홀더
- **왜 보나**: 대상 토큰(예: PEPE, ARB, SOL 등) 홀더 상위 20명의 움직임이 그 토큰 가격에 즉시 반영.
- **수집 방법**: Etherscan `tokenholderlist` 엔드포인트 (Pro 전용 일부) 또는 프로젝트별 홀더 스냅샷을 주기적으로 받아 상위 N명 추출.
- **운영 전략**: 주요 토큰 10~20종에 한정. 컨트랙트/락업 주소는 제외 플래그로 필터.

## 2. 데이터 소스 선정

| 용도 | 1순위 | 비용 | 비고 |
|---|---|---|---|
| ETH·ERC20 트랜잭션 조회 | **Etherscan API v2** | Free 3 req/s, 100K/day | 60+ 체인 단일 키 |
| L2 (Arbitrum, Base 등) | Etherscan API v2 같은 키 | 동일 한도 공유 | chainid 파라미터로 분기 |
| BSC/Polygon | Etherscan v2 (BscScan 통합) | 동일 | - |
| Solana | **Solscan Public API** | Free 100 req/min (v2) | 별도 구조 |
| 라벨 시드 | brianleect/etherscan-labels GitHub | 무료 | snapshot이라 수동 검증 |
| 크로스체인 분석 | **Dune API** (옵션) | 무료 credit 존재, 추후 검토 | 공개 쿼리 실행만 |
| 가격·시총 보강 | CoinGecko Free (기존 유지) | 10~30 req/min | 현행 유지 |

**적용 불가/보류**
- Nansen: 데이터 품질은 최고지만 기본 월 $150+. 유보.
- Arkham API: 공식 공개는 제한적이고 ToS 모호. 수동 활용만.
- DeBank: 개별 지갑 포트폴리오 조회에 좋지만 API 요금제 불투명. 보조 용도.

## 3. 비용 모델

Etherscan Free 기준 100K calls/day가 상한. 필요 호출량 계산:

- 감시 주소 N개 × 폴링 주기 T분 × 하루 24×60/T = 하루 N × (1440/T) 호출
- 예시: N=200 주소, T=15분 주기 → 200 × 96 = 19,200 calls/day (한도의 20%). 여유.
- Solana는 Solscan 별도 한도로 분리 관리. 초기 N=30 주소, T=15분 → 2,880 calls/day.
- CoinGecko Free는 현행대로 (가격 enrich 시에만).

**고정 운영비 목표: $0/월** (GitHub Actions + Google Sheets + 무료 API 조합).

## 5. 시그널 규칙 — MVP 5개

모두 "감시 지갑 활동에서 파생되는 관찰 가능한 이벤트"여야 한다.

1. **cex_outflow_spike**: 특정 CEX의 단일 주소에서 24h 내 순유출이 직전 7일 평균의 3σ를 초과. token별로 산정.
2. **cex_inflow_spike**: 위 반대 케이스. 매도 압력 예측.
3. **cold_to_hot_transfer**: 같은 CEX의 cold→hot 이동이 평균 이상으로 발생 (매도 준비 신호).
4. **smart_money_accumulation**: smart_money 카테고리 지갑 ≥3개가 같은 토큰을 24h 내 순매수.
5. **token_whale_concentration_shift**: 특정 토큰 상위 10 홀더의 합계가 직전 대비 ±2% 이상 변동.

각 규칙은 (조건, 임계값, 윈도우) 파라미터가 YAML(`config/signals.yaml`)로 빠져 있어 튜닝만으로 동작 변경 가능해야 한다.

## 6. 신규 파이프라인 (main.py 재구성)

```
1.  load_config + sheets init
2.  load watched_addresses (enabled=true)
3.  per chain:
      - pull last N hours of txs for each address (Etherscan/Solscan)
      - enrich with USD using CoinGecko
      - append to address_activity (dedup by tx_hash + direction + watched)
4.  SignalEngine.run(address_activity[recent]) → signals[]
5.  append to signals tab
6.  rank signals by severity+score → top K (예: 5~7)
7.  ClaudeAnalyzer.generate_brief(signals=top_signals) → brief_text
8.  save_daily_brief
9.  telegram_bot.send_daily_brief(brief_text)
10. log_run
```

기존의 "whale tx → filter → AI score → brief" 흐름이 "watched address activity → rule signals → AI narrative → brief"로 바뀐다. Claude는 스코어링이 아니라 **해석자 역할**만 한다.

## 9. 성공 기준

- **기능**: Whale Alert 호출이 코드베이스에서 0건 (`grep -ri whale_alert src/ scripts/` empty)
- **비용**: 월 $0 유지 (Etherscan Free + Solscan Free + CoinGecko Free + Google Sheets)
- **품질**: 시그널 생성이 일 평균 5~15건, 3주차부터 precision 60%+ (가격 방향성 일치)
- **차별성**: 생성된 브리프가 Whale Alert 공식 Twitter/Telegram에서 복제 불가한 내용(규칙 이름, 엔티티 단위 누적, 스마트머니 공통 움직임)을 1개 이상 포함
