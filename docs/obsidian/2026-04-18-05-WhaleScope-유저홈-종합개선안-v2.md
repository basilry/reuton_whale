## 2. 이슈 #1 — InsightsSidebar 스크롤 스파이 + 부드러운 스크롤 + 설정 제거

### 2.1 요구 분해

1. 사이드바의 각 섹션 링크 클릭 시 **브라우저 기본 점프가 아니라 smooth scroll**.
2. 스크롤 위치에 따라 해당 섹션의 링크가 **활성 색상**(이미 `.link[data-active="true"]` 가 정의돼 있음) 으로 표시.
3. 스크롤 이동 시 활성 상태가 **부드럽게** 다음 링크로 전이 (색상 자체가 CSS transition 을 타야 한다).
4. 현재 비활성화된 **"설정" 항목 제거**.

### 2.2 설계 선택지와 트레이드오프

| 접근 | 장점 | 단점 | 채택 |
|---|---|---|---|
| A. CSS `scroll-behavior: smooth` + JS scrollspy (IntersectionObserver) | 표준, 의존성 0, SSR 안전 | 클라이언트 상태 필요(`useState`) | ✅ |
| B. `react-scroll` 라이브러리 | API 간단 | 번들 ~10KB, 직접 `<Link>` 대체 필요 | ❌ |
| C. `scroll-spy` hash 동기화 (`scrollTo` + `setInterval`) | 간단 | 성능 나쁨, 스크롤 이벤트 throttle 필요 | ❌ |
| D. Anchor 클릭만 smooth, 활성 상태는 수동 | 빠른 구현 | 요구 #2·#3 미충족 | ❌ |

**결론**: A 채택. `"use client"` 내부에서 `IntersectionObserver` 로 각 섹션 관찰 → 가장 상단에 보이는 섹션의 id 를 `activeId` 상태로 유지 → 링크에 `data-active={href.endsWith(activeId)}` 바인딩.

### 2.3 활성 전환 "부드러움" 정의

사용자가 요구한 "부드럽게 다른 섹션 메뉴에 옮겨가야 해" 는 **색상 전환**으로 해석.

```css
/* insights-sidebar.module.css — 이미 존재하는 정의를 유지 */
.link {
  transition: var(--transition-bg), var(--transition-color);
  /* design-tokens.css 에 이미 정의:
     --transition-bg: background-color 160ms ease-out;
     --transition-color: color 160ms ease-out; */
}
```

활성/비활성 토글이 JS로 일어나면 transition 이 붙은 상태에서 부드럽게 전환된다. **추가 CSS 필요 없음** (이미 토큰 기반 transition 이 걸려 있음).

단, 접근성을 위해:

```css
@media (prefers-reduced-motion: reduce) {
  .link { transition-duration: 0.01ms; }
  html { scroll-behavior: auto; }
}
```

이 블록은 이미 `insights-sidebar.module.css:151-155` 에 `.link` 부분만 존재 → `html { scroll-behavior: auto; }` 는 `app/globals.css` 쪽으로 추가해야 한다.

### 2.4 구현 스텝

**Step 1 — `insights-sidebar.tsx` 를 클라이언트 훅 기반으로 변경**

```tsx
"use client";

import Link from "next/link";
import { useEffect, useState } from "react";
import styles from "./insights-sidebar.module.css";

const SIDEBAR_LINKS = [
  { label: "브리핑", href: "/#brief", id: "brief", icon: "article" },
  { label: "시그널", href: "/#signals", id: "signals", icon: "notifications" },
  { label: "감시 목록", href: "/#watchlist", id: "watchlist", icon: "visibility" },
  { label: "텔레그램", href: "/#telegram", id: "telegram", icon: "send" },
] as const;

// ⚠️ DISABLED_LINKS 는 전체 삭제 (요구 #1 마지막)

export function InsightsSidebar() {
  const [activeId, setActiveId] = useState<string>("brief");

  useEffect(() => {
    const sections = SIDEBAR_LINKS
      .map((link) => document.getElementById(link.id))
      .filter((el): el is HTMLElement => el !== null);

    if (sections.length === 0) return;

    const observer = new IntersectionObserver(
      (entries) => {
        // 화면에 교차 중인 섹션들 중 가장 위에 있는 것 선택
        const visible = entries
          .filter((e) => e.isIntersecting)
          .sort((a, b) => a.boundingClientRect.top - b.boundingClientRect.top);
        if (visible[0]) setActiveId(visible[0].target.id);
      },
      {
        // 상단 120px(TopNavbar 높이) 지점을 기준으로 교차 판정
        rootMargin: "-120px 0px -60% 0px",
        threshold: 0,
      },
    );

    sections.forEach((s) => observer.observe(s));
    return () => observer.disconnect();
  }, []);

  return (
    <aside className={styles.sidebar} aria-label="인사이트 내비게이션">
      <div className={styles.brandBlock}>…</div>
      <nav className={styles.nav}>
        {SIDEBAR_LINKS.map((item) => (
          <Link
            key={item.href}
            href={item.href}
            className={styles.link}
            data-active={activeId === item.id}
            aria-current={activeId === item.id ? "location" : undefined}
          >
            <span className="material-symbols-outlined" aria-hidden="true">{item.icon}</span>
            {item.label}
          </Link>
        ))}
      </nav>
    </aside>
  );
}
```

**Step 2 — `app/globals.css` 에 smooth scroll 추가**

```css
html {
  scroll-behavior: smooth;
  scroll-padding-top: calc(var(--inset-navbar) + var(--space-md));
  /* inset-navbar 는 디자인 토큰에 이미 정의됨. fixed 헤더 높이를 상쇄. */

@media (prefers-reduced-motion: reduce) {
```

`scroll-padding-top` 이 있어야 `#brief` 앵커로 점프할 때 헤더에 섹션이 가리지 않는다.

**Step 3 — `app/page.tsx` 의 섹션 id 감사**

```
id="brief" ✅ hero/brief 카드 래퍼에 반드시 존재해야
id="signals" ✅
id="watchlist" ✅

```

없는 id 는 섹션 래퍼 `<section id="...">` 로 추가. 감사 대상 라인:

- signalsPanel → `id="signals"`
- watchlist 카드 → `id="watchlist"`
- telegram CTA → `id="telegram"`

### 2.5 검증 체크리스트

- [ ] 4개 링크 클릭 시 각 섹션으로 smooth scroll.
- [ ] 스크롤 중 활성 링크가 색상 전환(160ms ease-out) 을 타며 바뀐다.
- [ ] 최상단일 때 `브리핑`, 최하단일 때 `텔레그램` 이 active.

- [ ] `aria-current="location"` 이 스크린리더에 정확히 전달된다.
- [ ] "설정" 링크 자체가 DOM에 없음.

## 3. 이슈 #6 — 텔레그램 연결 버튼을 모달(QR + 주소 복사)로

### 3.1 현재 상태

`app/page.tsx` 의 telegram 섹션에는 현재 "텔레그램 연결하기" CTA 가 존재하지만, 실제로는 봇 딥링크가 아니라 `href="#watchlist"` placeholder 상태다. 이번 개선에서 이를 실제 봇 링크 + QR 모달로 교체한다.

### 3.2 설계

**모달 내용**:
1. 제목: "텔레그램 봇 연결"
2. QR 코드 (봇의 딥링크 URL 을 인코딩) — 모바일에서 바로 카메라로 스캔해 대화 시작
3. 봇 딥링크 문자열 (예: `https://t.me/WhaleScopeBot?start=<nonce>`)
4. **주소 복사 버튼** — 클립보드로 딥링크 복사. 성공 시 tooltip/toast "복사되었습니다".
5. 부가 설명: "Telegram 앱에서 링크를 열거나, QR을 스캔하세요." + 옵션: "웹으로 열기" (target="_blank").
6. 닫기 버튼 + ESC/배경 클릭 닫힘.

### 3.3 컴포넌트 선택

| 옵션 | 판단 |
|---|---|
| 자체 `<dialog>` 엘리먼트 | 접근성 자동 처리(Modal focus trap, ESC close). Next.js SSR 호환. **채택** |
| `@radix-ui/react-dialog` | 이미 의존성이 있으면 채택. 없으면 추가 설치 부담 |
| shadcn `Dialog` | Radix 래퍼. 일관성 있음 |

→ 프로젝트에 이미 Radix/shadcn 의존성이 있으면 그걸 사용, 없으면 네이티브 `<dialog>` 기반으로 최소 구현.

### 3.4 QR 생성 방법

- **서버 렌더링 경로**: `qrcode` npm 패키지 (`qrcode.toDataURL`) 사용 → `<Image>` 로 렌더.
- **클라이언트 렌더링 경로**: `qrcode.react` → `<QRCodeSVG value={link} />`.

딥링크에 포함할 **nonce**(1회용 연결 토큰) 는 서버에서 생성해야 한다(자세한 인증 스킴은 별도 기획). MVP에서는 고정 봇 URL 만 써도 무방.

```tsx
// components/telegram-connect-modal.tsx
"use client";

import { useState, useRef, useEffect } from "react";
import { QRCodeSVG } from "qrcode.react";

interface Props {
  open: boolean;
  onClose: () => void;
  deepLink: string; // 예: https://t.me/WhaleScopeBot?start=XYZ
}

export function TelegramConnectModal({ open, onClose, deepLink }: Props) {
  const dialogRef = useRef<HTMLDialogElement>(null);
  const [copied, setCopied] = useState(false);

  useEffect(() => {
    const d = dialogRef.current;
    if (!d) return;
    if (open && !d.open) d.showModal();
    if (!open && d.open) d.close();
  }, [open]);

  async function handleCopy() {
    try {
      await navigator.clipboard.writeText(deepLink);
      setCopied(true);
      setTimeout(() => setCopied(false), 2000);
    } catch {/* 토스트로 실패 알림 */}
  }

  return (
    <dialog ref={dialogRef} onClose={onClose} aria-labelledby="tg-title">
      <header>
        <h2 id="tg-title">텔레그램 봇 연결</h2>
        <button onClick={onClose} aria-label="닫기">×</button>
      </header>
      <div className="qr">
        <QRCodeSVG value={deepLink} size={192} level="M" />
      </div>
      <p className="hint">Telegram 앱에서 링크를 열거나, QR을 스캔하세요.</p>
      <div className="linkRow">
        <code>{deepLink}</code>
        <button onClick={handleCopy} aria-live="polite">
          {copied ? "복사됨" : "주소 복사"}
        </button>
      </div>
      <a href={deepLink} target="_blank" rel="noreferrer" className="openWeb">
        웹으로 열기
      </a>
    </dialog>
  );
}
```

### 3.5 접근성 요건

- `<dialog>` 사용 시 focus trap 자동. 단, 모달 열릴 때 첫 포커스는 "주소 복사" 버튼 또는 닫기 버튼으로.
- QR 이미지는 장식적이므로 `aria-hidden="true"` + 텍스트로 딥링크를 제공.
- 복사 버튼 피드백은 `aria-live="polite"` 로 스크린리더에 전달.
- ESC 키로 닫힘 (native `<dialog>` 가 처리).

### 3.6 토큰 기반 스타일

별도 CSS Module 에 정의. 하드코딩 금지.

```css
.dialog {
  background: var(--surface-container);
  border: 1px solid var(--line);
  border-radius: var(--radius-lg);
  padding: var(--space-lg);
  max-width: 400px;
}
.dialog::backdrop {
  background: var(--scrim, rgba(0, 0, 0, 0.48));
  backdrop-filter: blur(4px);
}
/* ...etc */
```

`--scrim` 토큰이 디자인 토큰에 없으면 `design-tokens.css` 에 추가 (다크 테마용 별도 값).

### 3.7 검증

- [ ] 버튼 클릭 → 모달 열림.
- [ ] QR 스캔 시 봇 대화 진입 (모바일 테스트).
- [ ] 주소 복사 성공 토스트/상태 반영.
- [ ] ESC, 배경 클릭, 닫기 버튼 모두 동작.
- [ ] 모바일(393px)에서 모달이 화면을 벗어나지 않음, 내용 중앙 정렬.

## 4. 이슈 #7 — 리스크 카드에서 '마지막 실행 상태' 제거

### 4.1 현재 위치

`app/page.tsx:678` 라인:

```tsx
마지막 실행 상태: <strong>{humanizeRunStatus(data?.metrics.latestRunStatus)}</strong>
```

사용자 판단: "왜 거기 있는지 모르겠네." → **리스크 고지** 카드의 본질(투자 유의, 면책 안내)과 **운영 상태**(워커 실행 상태)는 정보 범주가 다름. 같은 카드에 섞이면 사용자가 리스크를 운영 문제로 오해할 수 있다.

### 4.2 조치

- 해당 라인과 주변 래퍼(`<p>` 혹은 `<div>`) 를 **제거**.
- `humanizeRunStatus` 함수는 **ops 페이지에서 계속 사용 중일 가능성** 있음 → 함수 자체는 **남겨두고 import/호출부만 제거**.
- `data?.metrics.latestRunStatus` 필드 참조도 유저홈에서는 제거.

### 4.3 대체? (선택)

사용자 요구는 "제거"이므로 대체하지 않는다. 단, 운영 상태를 정말 보고 싶다면:
- **ops 페이지**로 이동 안내 버튼을 별도 섹션에 작은 텍스트로. **본 스코프 아님, 별도 이슈로 분리**.

### 4.4 검증

- [ ] 리스크 카드에 "마지막 실행 상태" 문자열이 남아 있지 않음.
- [ ] 리스크 카드 레이아웃이 한 줄 제거 후에도 안정적(padding/gap 이 토큰 기반이면 자동 흡수).
- [ ] 운영자 페이지(`/admin`)의 동일 정보는 유지되는지 확인.

## 5. 이슈 #8 — 헤더 'Google Sheets 연결됨' → '데이터 연결'

### 5.1 현재

`app/page.tsx:454-463` 근처 hero/brief 상단에 `시장 인텔리전스` 타이틀 + `Google Sheets 연결됨` chip 이 존재.

### 5.2 판단

- "Google Sheets" 는 **내부 구현 디테일**. 일반 사용자에게 의미 없음.
- 근본적으로 이 프로젝트는 **Google Sheets 를 DW로 쓰고 있지만 Postgres/SQLite 등으로 이전 계획이 있음**(별도 Render 배포 가이드 문서 참조).
- **"데이터 연결" 로 단순화** + 연결 여부만 보여준다.

### 5.3 카피 최종안

| 상태 | 텍스트 | 칩 색상 토큰 |
|---|---|---|
| 연결 성공 | `데이터 연결됨` | `--good-soft` 배경, `--good` 텍스트 |
| 연결 실패/대기 | `데이터 연결 대기` | `--warn-soft` 배경, `--warn` 텍스트 |
| 완전 단절 | `데이터 연결 실패` | `--bad-soft` 배경, `--bad` 텍스트 |

### 5.4 구현

```tsx
// Before
<span className={styles.connectionChip} data-connected={state.sourceConnected}>
  Google Sheets 연결됨
</span>

// After
<span className={styles.connectionChip} data-status={
  state.sourceConnected ? "ok" : "waiting"
}>
  {state.sourceConnected ? "데이터 연결됨" : "데이터 연결 대기"}
</span>
```

CSS:
```css
.connectionChip[data-status="ok"]   { background: var(--good-soft); color: var(--good); }
.connectionChip[data-status="waiting"] { background: var(--warn-soft); color: var(--warn); }
.connectionChip[data-status="error"] { background: var(--bad-soft); color: var(--bad); }
```

### 5.5 왜 상세 이름(Postgres/Supabase 등) 을 쓰지 않는가

사용자가 읽기에 의미 없음. 또한 추후 데이터 원본이 바뀌어도 카피를 수정할 필요 없음.

### 5.6 검증

- [ ] 헤더에 "Google Sheets" 라는 문자열이 전혀 남아 있지 않음.
- [ ] 연결 실패 시 칩이 색상으로 경고를 전달.
- [ ] 스크린리더가 상태 변화를 `aria-live` 로 알림 (기존 동작 유지 또는 추가).

## 7. 디자인 일관성 가드

본 개선안 적용 시 아래 원칙을 반드시 유지:

1. **토큰 온리**. hex/rgba/oklch 리터럴 금지. 신규 색상 필요 시 `design-tokens.css` 에 토큰 추가 후 사용.
2. **AI-slop 금지 패턴**: side-stripe border, gradient text, `transition: all`, `ease-in` UI motion, 이모지 아이콘, `<div onClick>`.
3. **7-state 커버**: default/hover/focus/active/disabled/loading/error — 새 모달·버튼은 전부 포함.
4. **WCAG AA**: 대비 4.5:1 (본문), 3:1 (UI), focus-visible, `aria-live` 적절히.
5. **반응형**: 모바일(~393px) 먼저, `sm:` → `md:` → `lg:` 증강. 모달은 바닥 썸존 고려.

## 9. 오픈 이슈

- **Telegram 봇 딥링크의 nonce 생성**: 서버 측 엔드포인트 필요. 사용자당 1회용 토큰을 만들어 봇이 콜백으로 검증해야 함. MVP 는 정적 URL, 2차로 nonce 기반. → 별도 문서 필요.
- **데이터 연결 상태를 어느 레이어에서 판단하는가**: 현재 `loadInsightState` 의 try/catch 만으로 판단. 부분 실패(일부 테이블 불러오기 실패) 는 분리 필요. → 파이프라인 재설계 문서에서 다룬다.
- **사이드바 활성화가 `<h2>` 기준인가, `<section>` 기준인가**: 스크롤 위치가 섹션 전체를 기준으로 하되, 섹션 본문이 길면 오판할 수 있음. `rootMargin` 을 조정하며 QA 필요.
