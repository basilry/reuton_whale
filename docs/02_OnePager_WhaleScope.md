# One Pager: WhaleScope — AI 코인 고래 움직임 큐레이션 서비스

---

## 1. 이 도메인을 선택한 이유

크립토 시장은 정보 비대칭이 가장 극심한 금융 시장이다. 특히 "고래(대규모 보유자)"의 온체인 움직임은 시장 방향성의 선행지표로 알려져 있지만, 이 데이터를 해석하려면 블록체인 탐색기를 직접 읽고, 지갑 주소를 식별하고, 맥락을 파악해야 한다. 전문 트레이더가 아닌 일반 투자자에게는 사실상 접근 불가능한 영역이다.

"AI 요약/큐레이션" 도메인에서 이 주제를 선택한 이유는 명확하다:
- **데이터는 공개되어 있지만, 해석은 닫혀 있다** — 온체인 데이터는 누구나 볼 수 있지만, 의미를 읽어내는 건 소수만 가능
- **AI가 풀기에 적합한 문제** — 대량의 트랜잭션 데이터에서 패턴 추출 + 자연어 해설 생성은 LLM의 강점
- **뤼튼의 "Bring AGI Close to People" 미션과 정확히 일치** — 전문가만 이해하던 정보를 일반인도 소화할 수 있게 만드는 것

## 2. 기존 서비스들이 해결하지 못하고 있는 타겟과 문제

### 기존 서비스 현황

| 서비스 | 강점 | 한계 |
|--------|------|------|
| **Whale Alert** | 실시간 대규모 트랜잭션 알림, 20+ 체인 | Raw 데이터만 제공. "3,000 BTC가 바이낸스로 이동"이라고만 알려줌. 이게 매도 신호인지, 단순 지갑 정리인지 해석 없음 |
| **Arkham Intelligence** | AI 기반 지갑 라벨링, 엔티티 식별 | 영어 전용. UI가 복잡. 온체인 분석 경험 없으면 대시보드 해석 불가 |
| **Nansen** | 스마트 머니 추적, 수익률 분석 | 월 $150+. 기관/전문 트레이더 타겟. 일반 투자자 진입장벽 극심 |
| **코인업 (한국)** | 청산맵, 고래 알림 | 알림만 있고 맥락 해석 없음. 푸시 알림 피로도 높음 |

### 미해결 타겟과 문제

**타겟**: 코인에 투자하고 있지만, 온체인 분석은 할 줄 모르는 한국 일반 투자자 (2030 직장인 투자자)

**핵심 문제**: 
> "고래가 움직였다는 건 아는데, 그래서 나는 뭘 해야 하는지 모르겠다"

구체적으로:
1. **해석의 부재** — 기존 서비스는 "무슨 일이 일어났는지"만 알려주고, "왜 중요한지"는 말해주지 않음
2. **정보 과부하** — Whale Alert 텔레그램은 하루 수백 건의 알림. 중요한 건 5건인데 어떤 게 중요한지 모름
3. **언어 장벽** — 주요 분석 도구(Arkham, Nansen)는 모두 영어. 한국어 해설 서비스 부재
4. **비용 장벽** — Nansen 월 $150, Arkham Pro 유료. 일반 투자자에게 과도

## 3. 이 문제를 해결하기 위한 핵심 기능과 그 근거

### 서비스명: WhaleScope

**한 줄 정의**: 고래 온체인 움직임을 AI가 해석해서, 일반 투자자도 이해할 수 있는 한국어 브리핑으로 만들어주는 큐레이션 서비스

### 핵심 기능 3가지

#### 기능 1: 고래 움직임 AI 해설 (Whale Digest)
- Whale Alert API 등에서 대규모 트랜잭션 수집
- AI가 트랜잭션의 맥락을 분석: 거래소 입금(매도 가능성) vs 거래소 출금(장기 보유 신호) vs 지갑 간 이동(의미 낮음)
- **"3,000 BTC가 바이낸스로 이동했습니다"** → **"최근 7일간 이 지갑에서 3번째 거래소 입금. 과거 패턴상 대량 매도 직전에 이런 행동이 관찰됨. 주의 필요."**
- **근거**: 기존 서비스의 가장 큰 갭이 "해석의 부재". AI 요약/해석이 이 과제의 핵심 역량

#### 기능 2: 일일 큐레이션 브리핑 (Daily Brief)
- 하루 수백 건의 고래 알림 중 **진짜 중요한 5건만** 선별
- 중요도 판단 기준: 금액 규모, 반복 패턴, 시장 상황과의 연관성
- 매일 아침 텔레그램으로 한국어 브리핑 발송
- **근거**: 정보 과부하 해결. "매일 5분이면 고래 동향 파악 끝"이라는 사용자 가치

#### 기능 3: 내 관심 코인 맞춤 알림 (My Watchlist)
- 사용자가 관심 코인(BTC, ETH, SOL 등) 등록
- 해당 코인 관련 고래 움직임만 필터링해서 알림
- 알림 임계값 설정 가능 (예: 1,000 BTC 이상만)
- **근거**: 개인화 = 과제에서 요구하는 큐레이션의 핵심 요소. 모든 사용자에게 같은 정보가 아닌, 각자에게 의미 있는 정보만 전달

## 4. 이 피처의 성공을 판단할 수 있는 지표

### Primary Metric (북극성 지표)
- **일일 브리핑 열람률 (Daily Brief Open Rate)**: 발송된 브리핑 중 실제 열어본 비율
- 목표: 60% 이상 (일반 뉴스레터 평균 20~30% 대비 2배)
- 근거: "매일 열어보고 싶은 브리핑"이 되었다면 핵심 가치 전달 성공

### Secondary Metrics
| 지표 | 정의 | 목표 | 의미 |
|------|------|------|------|
| D7 Retention | 가입 7일 후 여전히 브리핑을 열어보는 비율 | 40%+ | 일회성이 아닌 습관적 사용 |
| 관심 코인 등록 수 | 사용자당 평균 등록 코인 수 | 3개+ | 개인화 기능 활용도 |
| 브리핑 공유율 | 브리핑을 다른 사람에게 공유한 비율 | 5%+ | 바이럴 계수, 콘텐츠 품질 대리 지표 |
| AI 해설 피드백 | "도움이 됐다/안 됐다" 반응 비율 | 긍정 70%+ | AI 요약 품질 검증 |

### 실패 판단 기준
- 2주 후 일일 브리핑 열람률 30% 미만 → 큐레이션 품질 또는 발송 타이밍 재검토
- 관심 코인 등록률 20% 미만 → 개인화 온보딩 UX 개선 필요

## 부록: 기술 구현 개요

### MVP 범위 (7일)
- **데이터**: Whale Alert API → 대규모 트랜잭션 수집
- **AI**: Claude API로 트랜잭션 맥락 해석 + 한국어 브리핑 생성
- **저장**: Google Sheets (Users, Transactions, Briefs)
- **인터페이스**: Telegram Bot (브리핑 발송 + 관심 코인 관리) + Streamlit (아카이브/대시보드)
- **스케줄링**: GitHub Actions (매일 오전 8시 브리핑 생성/발송)

### 향후 확장 (발표 시 언급)
- 실시간 알림 (중요도 HIGH인 움직임 즉시 발송)
- 거래소 유출입 비율 트렌드 차트
- 커뮤니티 센티먼트 통합 (트위터/텔레그램 감성 분석)
- 뤼튼 플랫폼 연동 가능성 (뤼튼 AI 서포터의 크립토 도메인 확장)
